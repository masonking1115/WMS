# Warehouse Management System — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a self-contained, desktop web WMS that prevents wrong-pallet picks, wrong-door loads, batch mixing, and over-capacity bays through scan verification, with route-optimized pick tickets, FIFO, live capacity tracking, audit logging, and role-gated screens.

**Architecture:** React + Vite + TypeScript SPA. Pure rules engine (`src/rules`) holds all business logic and is unit-tested with vitest. A reducer store (`src/store`) routes every mutation through the rules + audit logging and persists to `localStorage`. Screens (`src/screens`) are thin views over the store. Seeded with deterministic demo data.

**Tech Stack:** React 18, react-router-dom (HashRouter for GitHub Pages), Vite 5, Vitest 2, TypeScript 5.

---

## File Structure

- `src/types.ts` — domain types (User, Product, Batch, Pallet, Bay, StagingArea, DockDoor, Order, PickStep, AuditLogEntry, WarehouseState).
- `src/data/seed.ts` — deterministic seeded warehouse + `DEMO_TODAY`.
- `src/rules/rules.ts` — pure functions: lookups, capacity/bay rules, FIFO/expiration, route optimization, scan verification, pick-ticket derivation.
- `src/rules/rules.test.ts` — vitest coverage of every rule.
- `src/store/permissions.ts` — role → screen access map.
- `src/store/actions.ts` — discriminated-union Action type.
- `src/store/reducer.ts` — reducer; all moves + audit logging.
- `src/store/store.tsx` — context provider, localStorage persistence, reset.
- `src/components/` — Layout (sidebar nav), PageHeader, CapacityBar, Scanner.
- `src/screens/` — Login, Dashboard, PickTicket, Inventory, Loading, Unloading, Orders, Reports, AuditLog, Users.
